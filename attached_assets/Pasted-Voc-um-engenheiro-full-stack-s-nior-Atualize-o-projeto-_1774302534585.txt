========================
D) JOBS DE ATUALIZAÇÃO
========================

Crie server/jobs/scheduler.ts para disparar:
- Oferta do Dia: a cada 1 hora
- Relâmpago: a cada 5 minutos

Pode usar node-cron (recomendado) ou setInterval (MVP).
Precisa ter lock simples para não rodar duas execuções em paralelo para a mesma coleção (ex.: lock em memória ou flag no banco).

Algoritmo por execução da coleção:
1) Criar collection_batch (running, started_at=now)
2) Coletar todos os itens do snapshot atual
3) Inserir raw_collected_items (staging)
4) Atualizar membership:
   - Para cada item do snapshot:
     upsert membership set last_seen_at=now, is_active=true, last_batch_id=batch.id
   - Para itens dessa coleção que NÃO apareceram no snapshot atual:
     set is_active=false (imediato, conforme requisito do "ESGOTADO")
5) Finalizar batch (finished_at=now, status=success, contagens)
6) Disparar processamento dos raw items novos -> processed_items
7) Regras de triagem/publicação:
   - Se o card estiver completo E a marca estiver em whitelist (Nike, Adidas, Olympikus, Mizuno, Puma, Fila, Jordan, New Balance) => auto-aprovar e publicar offer
   - Caso contrário => triage_queue pending

Card completo mínimo:
- title (>=10)
- item_url válida
- image_url válida
- current_price > 0
- discount_percent entre 0 e 90 (se existir)

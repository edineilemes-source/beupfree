O caminho ideal seria em camadas:

1. Primeiro buscar no próprio Mercado Livre pelo ID do produto
Cada produto tem um ID tipo MLB-4189646883. Pela API do Mercado Livre, normalmente dá para obter atributos do item, categoria, variações, preço, fotos e descrição. A documentação oficial do Mercado Livre trata os anúncios como “itens” com atributos de produto.

2. Depois enriquecer com IA a partir do título + descrição + atributos
Exemplo:

Título: Tênis Asics Gel Revelation Masculino Preto Azul 42 BR
Descrição: Produto para caminhada, corrida e academia...

A IA preencheria:

{
  "marca": "Asics",
  "genero": "Masculino",
  "tamanho": "42",
  "modalidade": ["Caminhada", "Corrida", "Academia"],
  "cor": ["Preto", "Azul"],
  "tecnologia": "Gel",
  "idade": "Adulto"
}

3. Se ainda faltar dado, buscar fora do ML
Aí você pode pesquisar pelo nome do produto na web, no site da marca, em lojas como Centauro, Netshoes, Amazon, Shopee etc. Mas eu usaria isso como segunda etapa, porque é mais lento, mais caro e pode dar erro.

mas como eu usaria a IA pra enriquecer os dados dos produtos?

Você usaria a IA como uma etapa intermediária entre a coleta do produto e a gravação no banco.

Fluxo:

Produto vindo do Mercado Livre
        ↓
Título + preço + marca + descrição + atributos
        ↓
IA analisa o texto
        ↓
IA devolve JSON padronizado
        ↓
Você salva no Supabase
        ↓
Filtros do site usam esses campos

Exemplo prático:

Entrada para IA:

{
  "title": "Tênis Asics Caminhada Corrida Gel Revelation Masculino Preto/Azul 42 Br",
  "description": "Tênis esportivo indicado para caminhada, corrida e academia.",
  "brand": "Asics",
  "price": 94.04
}

Saída esperada da IA:

{
  "gender": "masculino",
  "age_group": "adulto",
  "sizes": ["42"],
  "colors": ["preto", "azul"],
  "modalities": ["caminhada", "corrida", "academia"],
  "technology": ["gel"],
  "confidence": 0.92
}

No banco, você teria colunas como:

detected_gender
detected_age_group
detected_sizes
detected_colors
detected_modalities
detected_technology
ai_confidence
enriched_at

A regra seria:

Se o produto ainda não foi enriquecido:
    manda para IA
    recebe JSON
    valida os dados
    salva no banco
Senão:
    usa os dados já salvos

O ideal é não chamar IA toda vez que o usuário abre o site. Você chama uma vez, salva o resultado e reaproveita.

Pedido que você pode mandar para o Replit:

Crie um serviço separado chamado product-enrichment-service.

Esse serviço deve receber title, description, brand, category e attributes do produto vindo do Mercado Livre e retornar um JSON padronizado com:

detected_gender,
detected_age_group,
detected_sizes,
detected_colors,
detected_modalities,
detected_technology,
confidence_score,
source_used.

Use IA apenas quando os dados não forem encontrados por regra simples. Primeiro aplique regex e dicionários locais. Depois use IA como fallback.

Salve o resultado no Supabase e faça os filtros laterais consumirem esses campos enriquecidos.

Essa abordagem deixa o BeUpFree mais inteligente e barato, porque a IA entra só quando realmente precisa.

Para o BeUpFree, eu não começaria usando IA para tudo. O melhor custo-benefício é uma arquitetura em 3 níveis:

Sem IA.

if(title.includes("Masculino"))
   gender = "Masculino";

if(title.match(/\b(3[4-9]|4[0-8])\b/))

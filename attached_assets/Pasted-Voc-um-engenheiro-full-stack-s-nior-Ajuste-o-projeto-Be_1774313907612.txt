2) ESGOTADO (REGRA OBRIGATÓRIA E ROBUSTA)
========================
- Itens que somem da lista devem continuar visíveis na seção, com selo "ESGOTADO".
- Implementar collection_memberships para controlar is_active por coleção.

Regra robusta para evitar "ESGOTADO por engano":
- Só marcar is_active=false quando o batch terminar com status=success.
- E só aplicar a desativação se total_collected >= MIN_EXPECTED_ITEMS (ex.: 30) para evitar falha parcial.

Algoritmo por batch (apenas no final e com success):
1) Para cada item visto: upsert membership set last_seen_at=now(), is_active=true, last_batch_id=batch.id
2) Marcar como esgotado: set is_active=false onde collection_source_id=... AND last_batch_id != batch.id

5) AUTO-APROVAÇÃO + TRIAGEM
========================
Implementar auto-aprovação (publicação automática) quando:
1) marca ∈ whitelist
2) card completo

Whitelist (normalizada):
- nike, adidas, olympikus, mizuno, puma, fila, jordan, new balance

Normalização:
- lowercase, remover acentos, pontuação, espaços extras
- "new balance" reconhecida com múltiplos espaços
- "air jordan" -> "jordan"
- Se detectar 2+ marcas no título (ex: "nike x adidas") => NÃO auto-aprovar (ambíguo)
- Se confiança < 0.85 => triagem

Card completo:
- title >= 10
- item_url válida http/https
- image_url válida
- current_price > 0
- currency = BRL (assumir BRL para ML Brasil)
- discount_percent coerente (0–90) quando existir

Pipeline:
raw_collected_items -> processed_items -> (auto-approve -> offers publicadas) OU (triage_queue pending)

Auditoria mínima em triage_queue:
- approved_by ('system'|'admin')
- auto_approved boolean
- auto_approved_reason jsonb
- brand_detected text
- brand_confidence numeric
- issues jsonb para pendentes

Não armazenar texto de comentários de reviews. Só rating_avg e reviews_count (números).
